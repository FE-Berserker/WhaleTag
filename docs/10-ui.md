# 10. UI 外观与主题

> 11 种主题模式(3 经典 + 8 策划)+ 8 套配色预设 + 快捷切换 + `'system'` 解析约束。

## 1. ThemeMode 联合类型

定义在 [src/renderer/reducers/settings.ts](../src/renderer/reducers/settings.ts)(**不是** presets.ts),即 settings slice 的 `themeMode` 字段。共 **11 个值**:

**经典(3)**:light / dark / system

**策划(8)**:`warm-paper` / `midnight-plum` / `frosted-mint` / `deep-ocean` / `dawn-blush` / `forest-ink` / **`soft-amber`** / **`high-contrast`**

策划主题通过 [src/renderer/theme/presets.ts](../src/renderer/theme/presets.ts) 的 `THEME_MODE_PRESET_MAP` 绑定固定的 `(mode, presetId)`:

| ThemeMode | mode | presetId | 渲染主色(浅 / 深) |
|---|---|---|---|
| `light` | light | `whale` | `#0ea5e9` |
| `dark` | dark | `whale` | `#818cf8` |
| `system` | dynamic | `whale` | 跟随 OS |
| `warm-paper` | light | `warm-paper` | `#b45309` |
| `midnight-plum` | dark | `midnight-plum` | `#c084fc` |
| `frosted-mint` | light | `frosted-mint` | `#14b8a6` |
| `deep-ocean` | dark | `deep-ocean` | `#38bdf8` |
| `dawn-blush` | light | `dawn-blush` | `#db2777` |
| `forest-ink` | dark | `forest-ink` | `#34d399` |
| `soft-amber` | light | `soft-amber` | `#d97706` |
| `high-contrast` | dark | `high-contrast` | `#fbbf24` |

> 注意:策划主题的"主色"上面列的是该主题**实际渲染**的颜色(已是 `mode` 对应侧的 `primary`,不是 preset 表里的浅 / 深双值)。

## 2. 解析与持久化

**`settings.themeMode`** 走 redux-persist。`useResolvedTheme(themeMode)` hook 返回 `{ mode: ResolvedThemeMode, presetId: string }`,其中 `ResolvedThemeMode = 'light' | 'dark'`(字面联合,非 `Exclude<...>`):

- 解析值**只派生,不持久化**(`'system'` 不会粘在退出时的明暗上)
- 经典 `system` 订阅 `matchMedia('(prefers-color-scheme: dark)')`,实时跟随 OS;策划主题不订阅(固定 mode)
- `safeMatchMedia` try/catch,jsdom / 测试环境回退 `'light'`

**`useResolvedThemeMode(mode)`** 是 `useResolvedTheme` 的便捷包装,只返回 `mode`(用于只需传给 `ExtensionViewPanel` 等消费者)。

### ⚠️ 关键约束:`'system'` 与策划主题模式绝不能直接流入 `createTheme.palette.mode`

MUI `createTheme` 只认 `'light' | 'dark'`,传其他值会抛对比度错误。防线:

1. **工厂签名收窄**:`createWhaleTheme(mode: 'light' | 'dark', presetId, fontSize = 13)` —— 漏解析是**编译错误**(`'light' | 'dark'` 字面类型)
2. **Root.tsx 解析**:`useResolvedTheme` 是唯一把这些原始模式转成实际 `(mode, preset)` 的地方

**易漏点**:`src/renderer/containers/MainLayout.tsx` 把主题传给 `ExtensionViewPanel`(prop 类型 `'light'|'dark'`,透传给扩展 iframe)。**必须** `useResolvedThemeMode(themeMode)` 后再传,否则:

- TS 编译错误(宽 `ThemeMode` 不能赋给窄 `'light' | 'dark'`)
- 扩展 iframe 收到 `'system'` 等字符串无法解析,在深色 OS / 策划深色主题下仍按浅色渲染

## 3. 底层配色预设

[src/renderer/theme/presets.ts](../src/renderer/theme/presets.ts) 维护 `PRESETS`(`whale` + 8 个策划)+ `THEME_MODE_PRESET_MAP`:

- **8 套预设**(`PRESETS` 数组):`whale`(默认)、`ocean` / `forest` / `sunset` / `mono` / `warm-paper` / `midnight-plum` / `frosted-mint` / `deep-ocean` / `dawn-blush` / `forest-ink` / **`soft-amber`** / **`high-contrast`** —— 实际 **12 项**(含 4 个老预设 `ocean / forest / sunset / mono`,它们仍在 `PRESETS` 数组中以备 themePreset 字段反向兼容,但**不再映射到任何 ThemeMode**)
- **`PresetPaletteVariant`** token 集:`primary` / `secondary` / `primaryLight` / `backgroundDefault` / `backgroundPaper` / `border` / `text` / `textSecondary` / `hover`
- 浅色共用 `LIGHT_NEUTRALS` slate 中性色;深色共用 `DARK_NEUTRALS` zinc 中性色;每套预设只改 `primary` / `primaryLight`
- `primaryLight` 深色可带 alpha(如 `#818cf820`),活动位置背景 16% 底色
- `getPreset(id)` 对未知 / undefined id 回退 `PRESETS[0]`(`whale`)

## 4. 入口

**设置 → 通用** `GeneralSection`:

- `ToggleButtonGroup` 11 个主题模式
- 选经典模式同步把 `themePreset` 设为 `'whale'`;选策划模式同步设为对应 preset id
- 移除独立"配色预设"选择器(避免双层选择混淆)

**`FileToolbar` 右侧** [ThemeQuickToggle](../src/renderer/components/ThemeQuickToggle.tsx):

- 单击按顺序循环 11 个模式
- 图标随当前态切;Tooltip 显示下一态名称(`switchTo*` 只覆盖 light/dark/system + 8 个策划的 i18n)
- `data-testid="theme-quick-toggle"`

## 5. 状态字段

- `themeMode: ThemeMode`(11 值,老值 `'light'/'dark'/'system'` 向后兼容)
- `themePreset: string` **保留但 Root 不读**(Root 直接读 `themeMode` 经 `useResolvedTheme` 派生 presetId);保留仅为了向后兼容历史 settings 与外部消费者
- 字段级迁移:`if (base.themePreset === undefined) base = { ...base, themePreset: DEFAULT_PRESET_ID }`
- 自定义 system prompt / load-user-settings 等 non-secret 设置走 `settings.ai*` 字段

## 6. i18n

`locales/{en,zh,zh-TW,ja,ko}/common.json` 扁平键:`light` / `dark` / `system` / `switchToLight` / `switchToDark` / `switchToSystem` / `theme` / `presetOcean` / `presetForest` / `presetSunset` / `presetMono` / `presetWarmPaper` / `presetMidnightPlum` / `presetFrostedMint` / `presetDeepOcean` / `presetDawnBlush` / `presetForestInk` / **`presetSoftAmber`** / **`presetHighContrast`**。

> 全应用 i18n 共 5 语言(en / zh / zh-TW / ja / ko);`scripts/check-locales.test.ts` 在 `npm test` 中强制各语言与 en 的 key 集合、复数形式(按 `Intl.PluralRules` 分类)与 `{{占位符}}` 对齐。复数后缀用 i18next v4 格式(`_one`/`_other`),ja/ko/zh* 只有 `_other`。

> ⚠️ **占位符名别撞 i18next 保留 option key**(`lng` / `ns` / `count` / `returnObjects` …)。例:`confirmTagLocation` 原用 `{{lng}}` 表示经度,撞 `lng`(语言覆盖)→ `t('confirmTagLocation',{count,lat,lng:<经度>})` 会把经度当语言码,崩 `TypeError: lng.toLowerCase is not a function`。**已改为 `{{lon}}`**(6 条:en ×2 + zh/zh-TW/ja/ko ×1)。`scripts/check-locales.test.ts` 加了守卫,禁止 `{{lng}}`/`{{ns}}` 占位符。

> 策划主题的 `switchTo*` 状态切换提示目前在 i18n 中**没有独立条目**,tooltip 显示下一态由 `ThemeQuickToggle` 直接读取预设 label。

## 7. 文件类型回退图标

[src/renderer/components/FileTypeIcon.tsx](../src/renderer/components/FileTypeIcon.tsx) + [src/shared/file-icon.ts](../src/shared/file-icon.ts):

- **39 类** `FileIconCategory`(不是 19):image / video / audio / pdf / word / excel / ppt / archive / **javascript / typescript / html / css / python / java / cpp / csharp / go / rust / shell / database / matlab / json / notebook / design / email / link / diskimage** / code / markdown / text / data / ebook / caj / drawio / excalidraw / font / model3d / executable / generic
- 前述文档(以及 `docs/06-thumbnails.md`)写"19 类"是简化版,常见文件覆盖即可;完整集合以 `file-icon.ts:24-63` 为准
- 大小写不敏感;多点名取最后一段;dotfile → `generic`
- Office 在 `whale-meta.OFFICE_EXT` 合并基础上按 word/excel/ppt 三分(`rtf→word`、`csv/tsv→excel`,仅图标)
- `model3d` 大类内部按品牌色细分:`ma / mb`(Maya 青)、`max`(3ds Max 蓝)、`skp`(SketchUp 红)、`c4d`(Cinema 4D 蓝)、`3dm`(Rhino 黑)、`ztl / zpr`(ZBrush 橙)、`sldprt / sldasm / slddrw`(SolidWorks 红,`PrecisionManufacturingIcon`)

**同类别内不同格式暂不细分**:JS / Python / Go 都是同一个青色 `Code` 图标,如需细分(扩展名文字角标),改 `FileTypeIcon.tsx` 叠定位 `Typography`。

## 8. 文件夹缩略图回归修复

[ThumbIcon.tsx](../src/renderer/components/ThumbIcon.tsx) 是所有视图(网格 / 列表 / 画廊 / Mapique / 日历 / 属性托盘)里文件夹缩略图的**唯一渲染路径**。原守卫 `if (entry.isDirectory || !canThumb) return` 把所有目录提前返回,导致 `loadFolderThumbnail` 死代码。改为只对**不可缩略的文件**短路(允许目录走 IntersectionObserver + doLoad)。

遗留:`firstThumbnailableFile` 不递归子目录,"只含子文件夹、无直接可缩略文件"的目录仍只显示纯图标(非 bug)。
